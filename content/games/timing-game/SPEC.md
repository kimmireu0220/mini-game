# 게임 아이디어: 시간 맞추기 게임

논의 정리 문서. 웹 멀티플레이어 게임으로 구현 예정.

---

## 1. 게임 개념

- **이름**: 시간 맞추기 게임 (부제·설명: 정해진 시간 예측해서 맞추기 / 온라인 게임 등)
- **규칙**: 목표 초(예: 5초)가 정해지면, **시작 신호 후 정확히 그 시간이 되는 순간**에 버튼을 누른 사람이 이김.
- **승자**: 목표 시점에 가장 가까운 시점에 누른 사람이 해당 라운드 승자.

---

## 2. 멀티플레이

- **방 생성**: 한 사람이 방을 만든다.
  - **방 이름** 선택(비어 두면 기본 제목 사용).
- **방 코드**: **6자리 숫자** (예: 482917). 말로 전달하기 쉽고, 입력이 빠르며, 100만 가지 조합으로 충돌 가능성 낮음. 링크 공유 시에는 `?code=123456` 형태로 포함 가능.
- **입장**: 방 코드 입력 또는 링크로 입장.
- **방 인원 제한**: 없음 (제한 없이 참가 가능).
- **진행**: 같은 방에 들어온 사람들이 **동시에 한 판** 진행. 모두 같은 시작 시점에서 같은 목표 초를 맞추는 대결.
- **호스트 나가면**: **방 종료** (방 폭파). 참가자들은 나가기만 가능.

---

## 3. 라운드

- **목표 시간**: 매 라운드 **랜덤**으로 설정. **범위: 5~10초** (5초는 감각, 7~10초는 세기 편한 구간, 밸런스 좋음).
- **시작**: **호스트만** "시작" 버튼을 누름. 방장이 시작하면 서버가 동일한 시작 시간을 모두에게 전달. **시작 직전 4, 3, 2, 1 카운트다운**(약 4초) 후 "시작!" 순간을 **공통 시작 시각**으로 사용해, 그 순간부터 각자 시간을 세고 버튼을 누름.
- **버튼 규칙**: 라운드당 **한 번만** 누르기. 조기 누르기(목표보다 일찍)는 실격 없이 **오차만** 표시해 순위에 포함. 라운드 종료 시까지 **안 누르면 기권** 처리.
- **종료 후**: 한 라운드가 끝나도 **방은 유지**, 다음 라운드를 **계속 반복**.

---

## 4. 플레이어·UI

- **플레이어 이름**: **닉네임** 사용.
- **닉네임 설정 시점**: **게임 첫 화면**. "방 만들기 / 방 들어가기" 선택 전에 닉네임 한 번 입력 → 로컬 저장 후 재사용, 진입이 빠름.
- **승리 횟수**: 각 플레이어(닉네임)별로 몇 번 이겼는지 화면에 표시.
- **결과 표시**: 라운드 종료 시 **모든 참가자**에게 동일하게 표시. 순위(1등, 2등…) + 각자 목표 대비 오차(몇 ms 빨랐는지/늦었는지). 예: 1등 닉네임A (+12ms), 2등 닉네임B (−30ms)…

---

## 5. 플랫폼·서버·배포

- **웹**: 지금 블로그/게임 페이지처럼 웹으로 제공.
- **실시간 서버**: **BaaS** 사용 권장. Supabase Realtime 또는 Firebase(Realtime DB / Firestore)로 방 생성·입장·시작 신호·누른 시각 동기화. 직접 WebSocket 서버 구축·운영 없이 빠르게 구현 가능, 스케일·장애 대응 부담 적음.
- **배포**: **블로그 페이지에 iframe**으로 게임 넣기 (기존 2048과 동일). 게임 진입 URL은 블로그 도메인 유지, 관리 일원화.

---

## 6. 미정·추가 논의 항목

- 방 이름은 "방 만들기" 화면에서 입력; 입장 시에는 방 코드만 사용.

---

## 7. 문서 이력

- 최초 작성: 논의 내용 정리 (방 이름 필수, 랜덤 목표 초, 라운드 반복, 승리 횟수 표시, 웹 플랫폼).
- 목표 시간 범위 확정: **5~10초** (추천안 반영).
- 게임 이름: **시간 맞추기 게임** (검색/유사 게임 스타일: 정해진 시간 예측해서 맞추기, 온라인 게임).
- 방 인원 제한: **없음** (제한 없이 참가 가능).
- 플레이어 이름: **닉네임** 사용. 닉네임 설정 시점: **게임 첫 화면** (추천안 반영).
- 방 코드 형식: **6자리 숫자** (추천안 반영).
- 라운드 시작 방식: **호스트만 시작**.
- 결과 표시: **전원** 순위 + 오차(ms). 모든 참가자에게 동일하게 표시.
- 서버·배포: **실시간 = BaaS**(Supabase/Firebase), **배포 = 블로그 iframe** (추천안 반영).
- 시작 전 카운트다운: **4, 3, 2, 1** 적용. "시작!" 순간 = 공통 시작 시각 (추천안 반영).
- 버튼 규칙: **라운드당 한 번만** 누르기. 조기 누르기는 **실격 없이 오차만** 표시 (추천안 반영).
- 안 누른 사람 = **기권**. 호스트 나가면 **방 종료**.
